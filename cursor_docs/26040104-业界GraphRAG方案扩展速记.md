# 业界 GraphRAG 方案扩展速记

> 目标：快速回忆“除了本项目实现外，业界还有哪些成熟路线与开源抓手”。

## 1. 先记框架：不是单一 Retriever，而是方案组合

企业级 RAG/GraphRAG 通常是组合能力：

- 数据治理（权限、脱敏、审计）
- 检索层（向量 + 关键词 + 图）
- 编排层（查询改写、路由、多跳、工具调用）
- 质量层（评测、可观测、回归）

速记：**工程化优先级 often > 算法新花样**。

## 2. GraphRAG 常见流派（对照学习）

### 2.1 KG-First（图谱优先）
- 先做实体关系抽取、融合，再检索问答。
- 优势：可解释、多跳友好、规则可控。

### 2.2 Vector-First + Graph Augmentation
- 先走向量召回，再用图关系做补强/扩展/重排。
- 优势：上线快，便于从传统向量 RAG 迁移。

### 2.3 Multi-hop Graph Retrieval
- 按路径逐跳找证据，适合因果链与跨文档关联问题。
- 成本：系统复杂度和延迟控制更难。

### 2.4 Community/Summary Graph
- 先做社区聚合与摘要，再按“全局 -> 局部”下钻。
- 优势：适合大规模语料下的全局问题。

## 3. 你问的“大厂成熟方向”一句话版

- OpenAI 风格：检索 + 工具调用 + 工作流控制，强调可控执行链。
- Microsoft/Azure 风格：权限治理、可审计、混合检索，强调企业上线稳定性。
- Google 风格：长上下文利用 + 分层检索 + 证据压缩，强调召回效率平衡。

速记关键词：**可控、可审计、可扩展**。

## 4. 开源项目学习抓手（可直接实践）

- `microsoft/graphrag`：社区摘要图、global/local 检索范式。
- `neo4j-labs/llm-graph-builder`：文档到知识图构建与图检索落地。
- `run-llama/llama_index`：同一框架下组合向量、图、Agent 工具流。
- `langchain-ai/langchain` + `langchain-ai/langgraph`：企业级编排与可观测生态。

## 5. 与你当前项目的映射

你当前 `neo4j-graphrag-python` 里的子类体系可对应：

- `VectorRetriever`：Vector-First 的基础召回层。
- `VectorCypherRetriever` / `HybridCypherRetriever`：图增强与后续联动层。
- 上层 `GraphRAG.search`：检索 -> 组装 -> 生成的统一入口。

一句话回忆：**你现在已在“可落地实现层”，下一步是补齐“方案架构层与评测治理层”。**

## 6. 下一步学习顺序（最省时间）

1. 跑通一个最小端到端（可复现实验）；
2. 加入混合检索与 fallback 策略；
3. 增加图扩展检索（Cypher 后处理）；
4. 建立离线评测集与回归机制；
5. 再做企业治理能力（权限、审计、可观测）。
